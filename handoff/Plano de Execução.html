Preparação do Ambiente

Antes de escrever qualquer linha de código

▼
🌐
Browser — supabase.com
Criar projeto no Supabase
Acesse supabase.com → New Project → nome "gaep" → anote a URL e a anon key
🌐
Browser — Supabase Dashboard
Rodar SQL de schema (001_schema.sql)
Dashboard → SQL Editor → cole todo o conteúdo de handoff/sql/001_schema.sql → Run
-- Cria: gaeps, operadores, categorias, atividades, -- relatorios, participantes, missoes, feriados, config_ia, audit_log
🌐
Browser — Supabase Dashboard
Rodar SQL de seed (002_seed.sql)
SQL Editor → cole handoff/sql/002_seed.sql → Run
Isso cria: GAEP-CAT, categorias, atividades, tipos de diária e config IA padrão
⌨️
Terminal — pasta do projeto
Configurar variáveis de ambiente
Crie/edite o arquivo .env.local com as chaves do Supabase
NEXT_PUBLIC_SUPABASE_URL=sua_url_aqui NEXT_PUBLIC_SUPABASE_ANON_KEY=sua_anon_key_aqui SUPABASE_SERVICE_ROLE_KEY=sua_service_key_aqui OPENAI_API_KEY=sua_openai_key_aqui
⌨️
Terminal
Copiar design tokens para o projeto
Cole handoff/tokens/design-tokens.ts em lib/design-tokens.ts
Atualize o tailwind.config.ts com as cores do arquivo
⌨️
Terminal
Copiar logo para public/
Copie handoff/design/GAEP-Logo.png para public/gaep-logo.png
